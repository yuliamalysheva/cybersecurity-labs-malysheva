# SQL Injection Demo (вразлива та захищена версія, Python)

Ця програма демонструє роботу **SQL-інʼєкцій** та моделює механізм взаємодії з базою даних у вразливому й захищеному режимах.  
Реалізовано простий приклад застосунку з авторизацією та пошуком, у якому можна побачити, як неправильна обробка введення користувача призводить до SQL-інʼєкцій, а параметризовані запити повністю їх блокують.

Програма повністю консольна, автоматично створює тестову базу даних, підтримує два режими роботи (вразливий та безпечний) і не потребує додаткових налаштувань.

## Функціонал

* Створення тестової бази даних із таблицею користувачів. 
* Вразливі операції:
  * логін із прямим підставленням введення в SQL-запит;
  * пошук користувачів через `LIKE '%term%'`, який допускає інʼєкції.
* Захищені операції:
  * логін із параметризованим запитом (`WHERE username = ?`);
  * безпечний пошук через `LIKE ?`.
* Демонстрація дії інʼєкцій:
  * обхід авторизації (`admin'--`);
  * обходження умов WHERE (`' OR '1'='1--`);
  * витік усієї таблиці через пошук (`%' OR '1'='1--`).
* Порівняння результатів вразливої та захищеної версії:
  * на одному й тому самому вхідному значенні програма демонструє різні результати залежно від типу SQL-запиту.
* Обробка помилок:
  * відсутність очікуваних даних;
  * використання некоректного введення;
  * неможливість виконати інʼєкцію в захищеному режимі.

## Вимоги

* Python 3.x  
* SQLite (стандартний модуль `sqlite3`)  
* Стандартні бібліотеки Python (без зовнішніх пакетів)

## Встановлення
1. Переконайтесь, що у вас встановлено Python 3.x  
   Завантажити можна з [офіційного сайту](https://www.python.org/downloads/).
2. Завантажте файл програми `sql_demo.py`.
3. Запустіть програму:
    ```bash
    python sql_demo.py
    ```
4. Дотримуйтесь інструкцій у консолі.
